이름 : 권창현
분반 : 주간A
작성한 컴파일러 : VS2012
테스트 해본 컴파일러 : VS2012, VS2010

교수님 제 노트북에서의 최대해상도가 1366*768이라서 요구하신 1100*800 을 소화를 제대로 소화시키지 못하여 800*600 으로 프로그래밍을 하였습니다..!
가장 기본적인 요구하신 기능인 : ( 시작 : S, 정지 : T, 계산 : C, 재시작 : R, 종료 : Q ) 의 기능키가 들어가있으며, ( D : 디버그 ) 기능 하나를 추가하였습니다..!
프로그래밍을 하면서 Debug를 쓰면서 작업을 하면되지만, VS2012 에서의 문제인지 제 노트북의 문제인지 Debug를 한후, 디버그를 종료를 하면 종료하는 시간이
최소 1분~3분 이상의 시간이 걸려서 그 보다 빠른 방법으로, 윈도우 프로그램 안에 디버그 기능을 넣게 되었습니다.
해당 디버그에는 각각의 초밥의 위치, 클릭되었는지 확인 하는 배열2개, 마우스의 X, Y 좌표, 게임이 시작된지, 디버그를 하는지를 디버그 하는 기능에 넣었습니다.

초밥을 생성시 색상은 rand 값을 이용하여 색상을 다양하게 변경하였습니다.
초밥을 클릭시 해당 초밥의 색상이 반전을 하는 방법을 사용하지 않고, 해당 초밥자체의 틀에 HS_DIAGCROSS 브러쉬를 사용하여 클릭된것인지를 확인할수 있게, 기능을 변경하였습니다.
초밥을 클릭하는 부분도 초밥이 놓여져있는 네모를 클릭하는것이 아닌, 해당 초밥의 도형을 클릭해야지 초밥이 클릭되게 기능을 변경하였습니다.
V.A.T 기능을 추가하여 구매한 금액의 10%는 V.A.T 붙어서 계산을 하는 기능을 추가하였습니다.
재시작키를 누르게되면, 바로 재시작이 되는것이 아닌 재시작을 할것인지에 대한 메시지박스가 나와서 키를 잘못눌러 재시작 하게 되는 부분을 보완하였습니다.
클릭했던 음식을 다시클릭하면 이미 선택한 음식이라는 메시지 기능을 추가하였습닏.
음식은 최대 10개까지만 먹을수 있게 제한을 두었습니다.

소스를 보시면 아시겠지만, 상당히 기능에 비해서 코드가 긴것을 아실 수 있습니다.
머리가 안되면 몸이 고생하듯이, 제 프로그램 실력이 조금 낮아서, 직접 하나하나 그려가며 작업을 하였습니다.
그로 인하여 초밥을 각 싸이드에 그리는 부분만 95줄정도가 됩니다..!
그외 초밥을 회전하기 위하여 초밥데이터 배열에 한칸 앞에씩 땡겨오는 부분도, 수작업으로 작업을 하였다가,
규칙을 알아내어 좀더 간단하게 코드를 수정하였습니다..!
