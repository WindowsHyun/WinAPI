이름 : 권창현
학번 : 2012180004
학과 : 게임공학과

제작된 프로젝트 버젼 : VS2012

게임 설명및 제작 방법 : 
게임을 실행시 기본적으로 Easy 모드로 자동 시작을 하며, 시간과 지뢰의 갯수등은 
Hint 메뉴를 클릭후 서브메뉴를 클릭을 하게되면 메시지 창으로 해당 지뢰의 갯수와, 남은 지뢰의 갯수, 찾은 지뢰의 갯수가 나옵니다.

게임 모드에 따라서 Easy 모드 : 9*9, Nomal 모드 : 16*16. Hard 모드 : 16*30
크기로 판의 크기가 분류가 되며, 지뢰의 갯수는 각각의 상황에 따라 랜덤하게 부여를 하게 만들었습니다.

왼쪽 버튼을 누를경우 지뢰가 없을경우 재귀함수를 통하여 해당 눌러진 위치에서의 8방향을 검사하여
숫자가 있는곳까지 칸이 열리게 하였으며, 칸을 벗어나거나, 이미 클릭된곳은 예외처리를 하였습니다.

지뢰의 주변에 숫자를 하여금 지뢰가 몇개 있는지 확인을 할때 사용 하는 알고리즘은
지뢰를 임의의 위치에 놓을시 해당 지뢰의 주변 8방향에 숫자 +1을 하고 다른 지뢰가 +1된 위치에 지뢰를 놓게 된다면
해당 +1의 값이 +2로 올라가는 방식으로 8방향의 숫자를 만들어서, 왼쪽버튼이 클릭과 동시에 숫자 판단을 하는것이 아닌,
미리 위치가된 숫자를 판단하여 왼쪽버튼을 누를시 해당 지뢰전의 숫자까지 찾아가는 알고리즘을 제작하였습니다.

각각의 모드에따라 미리 지정해둔 isDrawEasy[9][9] isDrawNomal[16][16] isDrawHard[30][16] 의 배열을 불러오는 방식으로
메모리를 조금 더 많이 사용을 하였지만, 동적배열을 하지 않는이상 배열을 강제로 늘리지 못하여 위와 같은 방법을 사용하게 되었습니다.

게임 시작시간과 끝날시의 시간측정 방법은 Windows.h 에 포하되어있는 GetTickCount() 라는 함수를 이용하여, 시작 시간에 불러와 시간을 저장후
게임이 끝나거나 게임오버가 될시에 GetTickCount() 시간을 불러와 두시간의 차이를 빼서 나누기 1000을 하였습니다.
시스템 자체의 시간이 ms로 사용을 하기에 1초에 1024ms 이지만 계산을 편하게 하기위하여 1000으로 나누기를 하였습니다.

이미지와 같은 리소스중 일부는 C:\Program Files\Microsoft Games\Minesweeper [윈도우7 기준] MineSweeper.dll 에 포함되어있는 이미지를 가져와
편집하여 사용을 하였으며, 그 외 이미지들은 인터넷에서 올려놓은 이미지를 사용하였습니다.

게임 상단의 START모드는 직접 이미지를 제작으로 하여 비트맵에 추가를 하였고, Rect를 이용하여 클릭할수있는 범위를 직접 만들어서 클릭을 할수있게 하였습니다.
그외 지뢰의 남은 갯수나, 시간초 부분은 TextOut을 이용하여 출력을 하였고, 크기와 색상은 SetBK 와 FontCreater를 이용 하여 색상과 크기를 조절하였습니다.
